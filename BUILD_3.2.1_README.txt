WHAT CHANGED
------------

FIX 1 — Boolean casing (helpers.py)
  Problem: clean_row() wrote "True" / "False" (mixed case) to Google Sheets.
  Google Sheets expects "TRUE" / "FALSE" (all caps).
  Fix: clean_row() now writes "TRUE" if truthy, "FALSE" if falsy.
  Also added a comment to parse_row() confirming it accepts both casings
  for backwards compatibility with any older sheet data.
  File: preferredhome_api/utils/helpers.py

FIX 2 — df_to_sheet() data loss risk (sheets_storage.py)
  Problem: df_to_sheet() called ws.clear() then ws.append_rows().
  If the app crashed between those two steps, the sheet was permanently wiped.
  This caused real data loss in Builds 3.1.15.1 and 3.1.15.2.
  Fix: Safe backup/restore pattern added to df_to_sheet():
    1. A full snapshot of the current sheet is taken before any changes.
    2. The write proceeds (clear + append).
    3. If the write fails at any point, the snapshot is restored automatically.
  Also added _bool_to_sheet_str() helper that converts Python booleans
  and mixed-case "True"/"False" strings to "TRUE"/"FALSE" at write time.
  This ensures boolean casing is correct even when clean_row() is bypassed.
  File: preferredhome_api/storage/sheets_storage.py

FIX 3 — _NUMERIC_FIELDS import (main.py) — ALREADY CORRECT, NO CHANGE NEEDED
  This was fixed in Build 3.1.15.5.
  main.py already uses: _NUMERIC_SET = set(cfg.NUMERIC_FIELDS)
  Confirmed correct. No change made.
