WHAT CHANGED
------------
Bug: add_listing() wrote the generated ID to row["ID"] (uppercase), but the
LISTINGS_COLUMNS list uses lowercase "id" to match the Google Sheet header.
This caused the id column to be blank on every new row.

The write sequence is:
  1. ws.clear()       — sheet is wiped
  2. ws.append_rows() — new data is written

Because the id was going into a key that didn't match any column, the row
structure was malformed on write. On some calls this caused an exception AFTER
the clear had already run, leaving the sheet empty.

Fix: Changed row["ID"] to row["id"] in add_listing() — one character.

This is the only change in this hotfix.
