WHAT CHANGED — 3 BUGS FIXED
-----------------------------

BUG 1 — delete_listing() used df["ID"] (uppercase) — FIXED
  sheets_storage.py line: mask = df["ID"].astype(str) == str(listing_id)
  Fixed to:              mask = df["id"].astype(str) == str(listing_id)
  Effect: Delete was throwing KeyError on every attempt. Now works correctly.

BUG 2 — listings_put() passed sheet_payload["ID"] (uppercase) — FIXED
  main.py was setting sheet_payload["ID"] = listing_id before calling update_listing().
  update_listing() iterates data.items() and skips k != "id" (lowercase only).
  The uppercase "ID" key was being written to the wrong sheet column on every edit.
  Fixed: Now removes any "ID" key and sets sheet_payload["id"] = listing_id.
  Also fixed in update_listing(): now skips k.lower() != "id" to catch both cases.

BUG 3 — listings_post() generated ID as "ID" (uppercase) — FIXED
  main.py was setting sheet_payload["ID"] = generate_id() before calling add_listing().
  add_listing() checks data.get("ID") or data.get("id") — fragile fallback chain.
  Fixed: Now always generates and passes lowercase "id" key for consistency.
